.sillok: A Korean-Specific Lossless Text Encoding Format for Linguistically-Informed Compression and Long-Term Archival

J. Suh
Preprint  ·  2026
Preprint

Overview

Korean text is conventionally stored in Unicode (UTF-8 or UTF-16), encoding each Hangul syllable block as a single code point. This is a reasonable general-purpose encoding, but it discards the internal phonological structure of each syllable: a Hangul block is composed of an initial consonant (choseong), a vowel (jungseong), and an optional final consonant (jongseong). Treating the block as atomic means that compression schemes cannot exploit this sub-syllable regularity.

.sillok is a lossless encoding format designed around Korean's actual phonological structure. It represents text at the jamo (phoneme) level, applying linguistically-motivated compression priors based on Korean phonotactics and morphology. The result is a format that achieves better compression ratios than UTF-8 on Korean text whilst remaining fully lossless and designed for long-term archival stability.

Design principles

The format takes its name from the Joseon Wangjo Sillok (Annals of the Joseon Dynasty), a centuries-long archival project that prioritised longevity and fidelity above all else. The encoding is designed with the same priorities: reproducibility, independence from any particular software ecosystem, and the ability to decode correctly without access to format documentation beyond the specification itself.

The compression model exploits the highly constrained phonotactics of Korean: not all jamo combinations are legal, and the distribution of legal combinations is highly skewed. A linguistically-informed model over jamo sequences compresses significantly more efficiently than a general-purpose byte-level model.
